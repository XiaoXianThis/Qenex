# Qenex 个人体验目标（收窄版）

> **定位**：先满足个人使用，不追求产品化。在 **Web / VS Code / JetBrains**（共享 UI）上，把交互做到接近 Cursor。  
> **优先级**：交互与体验 ≫ 稳定性 / 分发 / 生态长尾。  
> **Git**：检查点 + 一键还原是刚需；模式名称不重要（默认继续用检查点语义，如 `snapshot`）。

确认来源：主目标 = Cursor 手感；Q4=A–E；Q5=Web 也算；Q6=检查点刚需。

---

## 本轮要做（In Scope）

按手感接近度排序。每项标 **AI**（可无人值守实现）或 **你**（需拍板 / 真机手感验收）。

### 1. 检查点 + 一键还原（Q6=D，底座）

| ID | 需求 | 谁 |
|----|------|-----|
| CP-1 | 会话有「检查点」心智：每轮改动可回到某状态；对外文案偏 Cursor（检查点/还原），少暴露 snapshot/worktree 术语 | **你**定文案口径；**AI**改 UI 文案 |
| CP-2 | Changes 条：默认收起；左侧文件数；右侧 Undo（丢弃本会话改动）/ Keep（收下改动） | **AI**打磨；**你**手感验收 |
| CP-3 | 消息级时间旅行：点用户消息 → 对话 + 文件回到该消息发送前；可改文案发 | **AI**补齐一致性；**你**真聊一轮验收「Agent 是否还记得」 |
| CP-4 | Diff：点文件能看改了什么（可读即可，不追像素级 Cursor） | **AI** |
| CP-5 | 去掉 Changes 盲轮询；改为可见时 + 回合结束 / Undo / Keep / 还原后刷新 | **AI** |

### 2. 审批像 Cursor（Q4=C）

| ID | 需求 | 谁 |
|----|------|-----|
| AP-1 | 短选项文案、「不再询问」、不挡主对话阅读流 | **AI**出稿+实现；**你**定最终语气 |
| AP-2 | 审批出现位置/节奏接近 Cursor（嵌在线程或紧贴工具调用） | **AI**；**你**对比 Cursor 截图/记忆反馈 |

### 3. 模式切换手感（Q4=D）

| ID | 需求 | 谁 |
|----|------|-----|
| MD-1 | Agent / Plan / Ask（或当前 Agent 等价 mode）好找、图标/文案好懂 | **AI** |
| MD-2 | 切模式反馈明确（当前是什么、切完立刻变） | **AI**；**你**用常用 Agent 点几下 |

### 4. 输入区（Q4=E）

| ID | 需求 | 谁 |
|----|------|-----|
| IN-1 | 附件（至少图片）发送 + 历史里还能看见 | **AI** |
| IN-2 | `@` 引用工作区文件/文件夹进上下文 | **AI**实现；**你**定「先文件路径，还是要内容预览」若有歧义再问 |
| IN-3 | `/` 斜杠命令菜单（有 commands 才显示） | **AI** |

### 5. 宿主

| ID | 需求 | 谁 |
|----|------|-----|
| HZ-1 | 以共享 UI 为准；`bun run dev` / VS Code / JetBrains 交互一致即可，不为单端做特殊产品化 | **AI**改 `packages/ui`；**你**在自己常用宿主上点验收 |

## 明确不做（本轮 Out of Scope）

- Marketplace / 正式发版 / 多平台安装包策略  
- Agent 矩阵全覆盖、Claude subagents、Registry 长尾  
- Token 统计 / CheckList 等占位面板（假功能先藏或删，不新建大功能）  
- 稳定性专项（TTL、多端进程清理矩阵、CI 冒烟全绿）— 仅修「挡你用手感」的 bug  
- 四种 Git 模式产品化教育；不扩散 inplace/worktree 心智，检查点语义优先  

---

## 建议执行顺序（给 AI 的队列）

1. **CP-5** 去轮询（立刻减烦）  
2. **CP-1 / CP-2 / CP-4** Changes + 检查点文案 + Diff  
3. **CP-3** 消息还原一致性（你验收失忆）  
4. **AP-1 / AP-2** 审批手感  
5. **MD-1 / MD-2** 模式切换  
6. **IN-1 → IN-3 → IN-2**（`@` 通常比 slash 重，可后做）

---

## 你需要 Push 的最小集合

每周只需这几件（其余丢给 AI）：

1. **拍板**：检查点相关按钮最终叫什么（Undo/Keep vs 还原/保留 等）  
2. **手测**：一条真实对话 → 改文件 → Changes → 消息还原 → 再问 Agent 是否还记得  
3. **对比**：审批、模式切换和 Cursor 差在哪，用一句话/截图反馈  
4. **定 `@` 范围**（做到文件路径引用即可，还是要贴文件内容）— 若默认「路径 + 让 Agent 自读」可不额外拍板
